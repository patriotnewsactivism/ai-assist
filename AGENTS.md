# AGENTS.md

## Development Commands
- **Start backend API only**: `npm run dev` (tsx server.ts)  
  Server runs on `http://localhost:5000`. Requires at least one API key in `.env` (DEEPSEEK_API_KEY, GEMINI_API_KEY, etc.).
- **Start frontend dev server**: In a separate terminal, run `vite` (or `npx vite`).  
  Vite proxies `/api` requests to the backend at `http://localhost:5000`.
- **Build frontend for production**: `npm run build-frontend` (vite build)  
  Outputs to `/dist`.
- **Start production server**: `npm run start` (executes `start.sh`)  
  Builds frontend then starts Express server serving static files from `/dist`.
- **Type-check TypeScript**: `npx tsc --noEmit`

## Environment Setup
- Copy `.env.example` to `.env` and fill in required API keys.
- Server fails to start if no API keys are provided (checked at startup).
- Optional API keys: GEMINI_API_KEY, ANTHROPIC_API_KEY, DEEPSEEK_API_KEY, OPENAI_API_KEY, GROQ_API_KEY, GITHUB_TOKEN, TAVILY_API_KEY.

## Architecture Overview
- **Backend**: 
  - `server.ts`: Express API server with endpoints for `/api/debate`, `/api/documents/*`, `/api/repo/*`, `/api/config`.
  - `engine/`: Contains the adversarial AI loop logic:
    - `routeInput(userInput)`: Classifies task into RESEARCH_MODE, DATA_MODE, or CODE_MODE using LLM/router.
    - `runAdversarialLoop(routing, input, maxIterations, progress?)`: Runs multiple rounds between LLM personas (Creator/Auditor variants) until "PERFECT" is output or max iterations reached.
- **Frontend**: React + Vite (`src/`):
  - `DebateInput`: Form to submit task and max iterations.
  - `DebateProgress`: Shows router classification and live Creator vs. Auditor panels.
  - `DebateResult`: Displays final output with tab for full debate history.

## Quirks & Conventions
- The adversarial loop uses stateful Auditor (critiques accumulate across rounds) and stateless Creator (fresh context each round).
- In CODE_MODE, the synthesizer output is parsed for code blocks and automatically built in a sandbox; build summary is fed to the Judge in subsequent rounds.
- File uploads and URL fetching are processed into text context for the AI loop.
- GitHub integration allows importing repo contents and creating pull requests with the final output.
- All API calls use a fallback chain (DeepSeek → Groq → Gemini → Anthropic) based on available keys.

## File & Directory Significance
- `engine/index.js`: Exports all engine functions (routeInput, runRoundtable, etc.).
- `src/types.ts`: Shared TypeScript interfaces between frontend and backend.
- `dist/`: Production-built frontend assets (generated by `npm run build-frontend`).
